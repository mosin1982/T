\# Global Language Support



\*\*Project:\*\* T

\*\*Maintainer:\*\* T Technology Research Lab

\*\*Status:\*\* Public Alpha / Research Software



T is designed as an English-first global public alpha project with planned multilingual documentation and dashboard support for international research users.



```text

Research only. Not financial advice.

```



\## 1. Purpose



The purpose of global language support is to make T easier to understand, evaluate, and use by researchers, developers, students, independent builders, and small teams across different regions.



T may be used by global GitHub visitors, not only Indian users. Clear language support improves:



\* Accessibility

\* Trust

\* Documentation clarity

\* Contributor onboarding

\* Client demo value

\* Donation and support understanding

\* Research safety communication



\## 2. Primary Language



The primary project language is:



```text

English

```



English remains the default language for:



\* Main README

\* GitHub repository structure

\* Code comments

\* CI messages

\* Developer documentation

\* Release notes

\* Technical architecture



This keeps the repository understandable for the global open-source community.

\## 5. Translation Safety Rule



Financial and research disclaimers must be translated carefully.



The required English disclaimer is:



```text

Research only. Not financial advice.

```



Translated disclaimers must preserve the same meaning:



\* Research-only use

\* No financial advice

\* No trading instruction

\* No return assurance

\* No advisory relationship

\* No service obligation through donation



\## 6. Documentation Translation Strategy



T should not add many weak translations at once.



Recommended approach:



```text

Step 1: English primary docs

Step 2: Hindi/Hinglish support notes

Step 3: Short translated README summaries

Step 4: Translated safety disclaimers

Step 5: Community-reviewed language files

Step 6: Dashboard language selector

```



\## 7. Suggested Folder Structure

\## 9. What Should Not Be Translated Automatically



The following should not be translated carelessly:



\* Legal-style disclaimers

\* Safety policy text

\* Donation notices

\* Professional service boundaries

\* Market compliance statements

\* Risk warnings



These should be reviewed manually before being published.



\## 10. Contributor Guidance



Language contributors should:



\* Preserve the research-only meaning

\* Avoid unsafe financial claims

\* Avoid trading instruction language

\* Avoid return-assurance wording

\* Keep translations simple and clear

\* Mention if a translation needs review



\## 11. Official Positioning



T should be described globally as:



```text

T is an English-first, global public alpha financial research project with planned multilingual documentation and dashboard support.

```



T should not be described as:



```text

A multilingual financial advisory system

A trading instruction platform

A return-assurance product

A fully localized regulated finance service

```
